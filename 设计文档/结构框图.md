## 总体分层框图

```text
+--------------------------------------------------------------+
|                           User / Web UI                      |
|--------------------------------------------------------------|
| - Natural language input                                     |
| - Mode confirmation                                          |
| - Task confirmation                                          |
| - Run / Agent / Evidence / Validation display                |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                             CLAW                             |
|--------------------------------------------------------------|
| Top-level governance and review layer                        |
|                                                              |
| - Task understanding                                         |
| - Mode review                                                |
| - Execution path review                                      |
| - System-state-aware audit                                   |
| - Task standardization                                       |
|                                                              |
| Control plane: Markdown                                      |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                   Router  (Layer 1.5)                        |
|--------------------------------------------------------------|
| Task decomposition & dispatch layer                          |
|                                                              |
| Input:  task_contract (normalized goal)                      |
| Output: WorkItems + RouteGroups + RouterDecision             |
|                                                              |
| - WorkItem splitting (rule-driven, conservative)             |
| - RouteGroup assembly (mode / isolation / dependency)        |
| - Bot assignment (ProjectMakerBot / KnowleverBot)            |
| - RouterDecision + RouterReviewResult serialized             |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                             Mode                             |
|--------------------------------------------------------------|
| Task paradigm definition                                     |
|                                                              |
| Examples:                                                    |
| - knowledge_mode                                             |
| - project_mode                                               |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                           Executor                           |
|--------------------------------------------------------------|
| Mode-specific execution brain                                |
|                                                              |
| Examples:                                                    |
| - knowledge_executor                                         |
| - project_executor                                           |
|                                                              |
| - Select main workflow                                       |
| - Dispatch runs                                              |
| - Coordinate results                                         |
|                                                              |
| Control plane: JSON                                          |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                       Workflow FSM Layer                     |
|--------------------------------------------------------------|
|  +---------------------------+   +-------------------------+ |
|  |      Main Workflow FSM    |   |   Shared Workflow FSM   | |
|  |---------------------------|   |-------------------------| |
|  | - Mode-level main flow    |   | - Reusable governed     | |
|  | - Task lifecycle control  |   |   workflows             | |
|  |                           |   |                         | |
|  | Examples:                 |   | Examples:               | |
|  | - KnowledgeExplainFSM     |   | - HumanActionFSM        | |
|  | - ProjectMainFSM          |   | - ManualReconcileFSM    | |
|  | - ProjectDebugFSM         |   | - ValidationFSM         | |
|  +---------------------------+   +-------------------------+ |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                              Run                             |
|--------------------------------------------------------------|
| Concrete execution instance                                  |
|                                                              |
| - run_id                                                     |
| - status                                                     |
| - logs                                                       |
| - evidence                                                   |
| - validation result                                          |
| - result                                                     |
+-------------------------------+------------------------------+
                                |
                                v
+--------------------------------------------------------------+
|                             Agent                            |
|--------------------------------------------------------------|
| Execution unit inside a run                                  |
|                                                              |
| - Handle subtask                                             |
| - Call skills                                                |
| - Apply patterns                                             |
| - Return evidence + result                                   |
+-------------------------------+------------------------------+
                                |
                                v
+-----------------------------------+    +---------------------+
|              Pattern              |    |        Skill        |
|-----------------------------------|    |---------------------|
| Reusable micro-procedure          |    | Concrete capability |
|                                   |    |                     |
| - checklist                       |    | - tool call         |
| - mini-FSM                        |    | - file operation    |
| - recipe                          |    | - command execution |
| - decision tree                   |    | - validation        |
| - rule pack                       |    | - network probe     |
+-----------------------------------+    +---------------------+
```
