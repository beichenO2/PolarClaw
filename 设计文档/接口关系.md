## 2. HTTP API 接口

### POST /api/tasks
```
Request  → { goal, mode?, constraints?, session_id? }
Response → { task_id, session_id, status, mode, created_at }
```
触发后台异步处理：normalize → router → dispatch_route_groups

### GET /api/tasks/{task_id}
```
Response → { task_id, status, mode, goal, run_id?, created_at, updated_at }
status ∈ { processing, done, done_with_issues, done_echo_fallback, failed, blocked, need_human }
```

### GET /api/tasks/{task_id}/result
```
Response → {
  task_id, run_id, status,
  router_decision,          ← RouterDecision
  router_review_result,     ← RouterReviewResult
  work_items[],             ← WorkItem[]
  route_groups[],           ← RouteGroup[]
  agent_result { outputs[], model_response, fact_status },
  validation_report {
    judgment,               ← PASS | FAIL | BLOCKED | NEED_HUMAN
    criteria_results[],
    violations[],
    summary,
    router_validation { work_items_valid, route_groups_valid, decision_traceable, judgment }
  },
  human_actions[],
  blocked_reason?
}
```

### GET /api/system/providers
```
Response → {
  coding_plan:    { available, key_present, model },
  minimax:        { available, key_present, model },
  qwen_dashscope: { available, key_present, model },
  echo:           { available, key_present, model },
  task_assignments: { coding, router, agent, vision, debug, general },
  active_agent_provider: "model-name"
}
```

## 3. Runtime Objects（对象契约）

### 数据流向

```
user input (goal)
    │
    ▼ normalize_task_input()
task_contract { task_id, goal, mode, constraints, context,
                editable_whitelist, acceptance_criteria,
                raw_input, normalized_input }
    │
    ▼ router.route()
WorkItem[]          ← 每个独立目标单元
RouteGroup[]        ← 兼容 WorkItem 的执行组
RouterDecision      ← 包含以上两者 + warnings + dispatch_ready
RouterReviewResult  ← 路由摘要
    │
    ▼ dispatch_route_groups() / run_task()
Run { run_id, status, logs, evidence, validation_result }
    │
    ▼
agent_result { outputs, model_response, fact_status }
validation_report { judgment, criteria_results, violations, router_validation }
evidence_pack { ... }
```

### 关键对象字段一览

| 对象 | 核心字段 |
|------|---------|
| `task_contract` | task_id, goal, mode, constraints, editable_whitelist, acceptance_criteria |
| `WorkItem` | work_item_id, task_id, goal, recommended_mode, priority, status, isolation_required, dependency_ids |
| `RouteGroup` | route_group_id, task_id, work_item_ids, mode, bot_name, status, blocking_reason |
| `RouterDecision` | task_id, work_items[], route_groups[], warnings[], dispatch_ready |
| `RouterReviewResult` | task_id, status, decomposition_summary, warnings[] |
| `RouteGroupRuntime` | route_group_id, status, current_stage, run_ids[], waiting_for, human_confirmation_required |
| `RouteGroupResult` | route_group_id, status, summary, result_ref, validation_report_refs[] |

## 4. 内部模块接口

### ModelGateway
```python
get_provider_for_task(task_type: str) -> ModelProvider
# task_type ∈ { coding, router, agent, vision, debug, general }

provider_status() -> dict  # 返回所有 provider 状态 + task 分配
```

### ModelProvider（抽象接口）
```python
generate(messages: list[dict], params: dict) -> str
health_check() -> dict
MODEL_NAME: str  # class attribute
```

**task_type → provider 映射：**

| task_type | model | provider | latency |
|-----------|-------|----------|---------|
| coding | qwen3-coder-plus | CodingPlan | ~1.8s |
| router | kimi-k2.5 | CodingPlan | ~2.4s |
| agent | kimi-k2.5 | CodingPlan | ~2.4s |
| vision | MiniMax-M2.7 | MiniMax | ~5.6s |
| debug | MiniMax-M2.7 | MiniMax | ~5.6s |
| general | qwen-plus | DashScope | ~2.9s |

### RuntimeStore
```python
# 任务生命周期
save_task(task_id, task_contract) / load_task(task_id)
update_task_status(task_id, status, run_id)
list_tasks()

# Run 产物
save_run_result(task_id, run_id, result)
save_evidence_pack(task_id, run_id, pack)
save_validation_report(task_id, run_id, report)

# Router 产物
save_router_decision(task_id, decision)
save_router_review_result(task_id, review)
save_work_items(task_id, items[])
save_route_groups(task_id, groups[])
save_route_group_runtime(task_id, rg_id, runtime)
save_route_group_result(task_id, rg_id, result)

# 聚合查询
get_full_task_result(task_id) -> dict  # 汇总所有产物
```

### PromptAssembler
```python
assemble(role, mode, task_context, packed_context) -> list[dict]
# 返回 OpenAI-compatible message list
```

### ValidatorEngine
```python
validate(task_contract, agent_result) -> validation_report_dict
validate_router(router_decision) -> router_validation_dict
# judgment ∈ { PASS, FAIL, BLOCKED, NEED_HUMAN, PARTIAL }
```

### Router 模块
```python
# backend/router/
splitter.split(task_contract) -> list[WorkItem]
grouping.group(work_items)    -> list[RouteGroup]
router.route(task_contract)   -> (RouterDecision, RouterReviewResult, list[RouteGroupRuntime])
```
